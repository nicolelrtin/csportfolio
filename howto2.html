html & javascript: calculator & to do list

the calculator: html

Building a functional calculator is relatively simple, but it requires an understanding of html and javascript, which may not come as easily.
First, a single input element was created, so the user can type in the first value of the equation. It should then be given a descriptive id (which function will be explained later).
Succeeding this is the dropdown in which the calculator functions are stored. There is no "dropdown" tag, but rather one called "select". Note that this tag is not self closing; each option will go inside the two tags. This should also be given an ID, such as "function". After the first select tag, "option" elements are used to identify the various functions. This tag is also not self closing; in-between is the label the user should be able to see. Once all the desired options are listed, the select tag can be closed. Following this should be a second input tag for the second number in the equation.
Next, we have a button that acts as an interactive equals sign. This part is crucial so the computer knows to use the code block written in the JavaScript. Luckily, creating this element is very easy! Simply write a "button" tag, give it an id, write a label for the user (outside the closing bracket), and close the tag. One version could potentally look like
<button id="btn"> calculate </button>.
The very last step is to create a space where the computer can input the answer to the equation. This can be done using a "span" tag. In this space, the computer can perform the above calculation and then store the answer for the user to see. This should also be given an ID tag.

the calculator: javascript

As per all javascript documents, the first step of setting up our calculator is to create a .js document and link it in our html. In the javascript document, we have to declare our variables. When creating the elements of the calculator in HTML, each element should have been given an ID. For simplicity, the name of these IDs will also be our variable names. Each of our variables will be assigned a DOM element document.getElementById(" ") , which essentially tells the JavaScript to use the element in the HTML with the corresponding ID. With these variables, we can perform calculations with each of their given values.
Next, we add an "Event Listener" to the button, which formulates a response when the button is "clicked". The function that it carries out is defined by the statements succeeding it.
Before we define the conditions that run when the button is clicked, we created new variables that is specific to this function. This helps to clean the code and make it easier to read overall. Because there are so many operating functions for the calculator to potentally perform, we use if/then statements to specify what happens in each operation. In this case, we use "if" for the first statement and "else if" fof the rest. Taking the first line of the first as an example, the JavaScript essentially reads, "if this function is true, run this code block". Note that the condition is denoted as "equal to" by two equals signs, and the open curly bracket holds the conditional code.
Inside is like simple math. "Result" is where the answer is stored, as characterized by the sum of ui1 and ui2. The uis are preceded by a plus sign to tell the computer that ui is a number and not a string. Without it, the computer would simply put the values next to each other (i.e. 2 + 3 = 23). The last line tells the computer to place the result in the HTML of the display.
